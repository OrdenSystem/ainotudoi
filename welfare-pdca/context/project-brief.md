# プロジェクトブリーフ — 障害福祉システム

## 元プロンプト
> 障害福祉システム開発が目的。環境ツールは、SalesforceとAppsheet、GASアプリ、GAS、DBにGCPからCloudSQLを使用する。
> 上記チーム構成で、PDCAサイクルを最低2周回す。

## 目的（一次解釈、初回サイクル開始前に必要なら追記する）
- 障害福祉サービス事業所の業務（利用者管理、サービス提供記録、請求、計画作成 等）を、Salesforce / AppSheet / GAS / GCP CloudSQL の組合せで支援する。
- 紙・Excel・分散SaaS の現状から、現場入力 → 集計 → 請求のフローをデジタルで一貫させる。

## 制約・前提
- 既存資産（AppSheet / Salesforce / GAS）は別途用意済み。本プロジェクトでは設計成果物のみ出力する。
- GCP は新規だが、別プロジェクトのコピーで構築する。コピー元仕様は `existing-assets.md` に記載。
- 個人情報保護法および障害者総合支援法・関連報酬告示の遵守が前提（具体の法務判断は専門家レビュー）。

## このプロジェクトで決めない事
- 個別事業所の運用ルール
- 帳票・印刷物のレイアウト
- 多言語・多通貨対応
- モバイルアプリのネイティブ開発（AppSheet で代替）

## サイクルゴール（指針）
- **Cycle 1**: 全体アーキテクチャと主要データモデル、連携経路、セキュリティ方針を **設計図ベース**で確定。
- **Cycle 2**: Cycle 1 の Critical / Major 指摘を反映し、運用観点・福祉業界要件・整合性で PASS を狙う。

## 用語ノート
- 「利用者」: 障害福祉サービスを受ける本人。Salesforce では Person Account ベースが候補。
- 「サービス等級・支給決定」: 受給者証に紐づく公的決定情報。
- 「サービス提供記録（ログ）」: 日々の支援内容。AppSheet/GAS で現場入力対象。
- 「請求」: 国保連レセプト等。本サイクルでは外部システム連携前提とし、内部データ準備までを設計。
