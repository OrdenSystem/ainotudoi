## §5. 投入手順（次フェーズ）

### Step 1: AppSheet Enum 値拡張
```bash
python scripts/appsheet-add-enum-value.py \
  --app-name HopeCareDX_ainotudoi-443914355 \
  --table 001_事業所加算マスタ \
  --column 事業所REF \
  --values 児童入所施設 短期入所 日中一時支援 \
  --apply
```

### Step 2: CSV データ準備
- `scripts/master-rows/児童入所施設_事業所加算.csv` (54 行 = 加算47 + 減算7)
- `scripts/master-rows/短期入所_事業所加算.csv` (48 行)
- `scripts/master-rows/日中一時支援_事業所加算.csv` (3 行)

### Step 3: AppSheet DB に行投入
```bash
python scripts/appsheet-add-records.py \
  --app-id b9e4f84d-f9b9-4376-97f1-83e3b07122e3 \
  --table 001_事業所加算マスタ \
  --csv scripts/master-rows/児童入所施設_事業所加算.csv \
  --apply
```

### Step 4: CloudSQL DDL 適用
```bash
psql ... -f welfare-pdca/implementation/ddl/09_市町村マスタ.sql
psql ... -f welfare-pdca/implementation/ddl/10_日中一時単価マスタ.sql
psql ... -f welfare-pdca/implementation/ddl/12_大和高田市_初期データ.sql
```

## §6. 残課題

| 課題 | 対応案 |
|---|---|
| 加算ごとの「適用条件」「必要書類」「加算適用示唆プロンプト」 | リタリコ既存行を参考に手動入力 or AI 自動生成 |
| 「利用者加算マスタ」の対応する加算 | `001_利用者加算マスタ` 側で同様に行追加（重複しない加算のみ） |
| R8.6 改定の処遇改善加算 適用日（2026-06-01〜） | `適用開始_年月日` 列に 2026-06-01 を一斉セット |
| 大規模減算の「定員 20 人以上」条件 | `加算内容・適用条件` 列に明記 |
